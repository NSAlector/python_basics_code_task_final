# python_basics_code_task_final

Вам необходимо разработать систему, позволяющую проверить и сохранить информацию об уникальных данных из набора (каталог .dataset).

Модуль io
Исходные данные могут быть представлены в виде:
 - файл изображения в формате .png, .jpg.
 - текстовый файл: первые три значения кодируют длину (M), ширину изображения (N), третье - количество компонент кодировки пикеля (K). Далее записаны MNK значений. При K = 3 кодировка пикселя и порядок следования значений представлены в формате RGB.

 Для работы с различными типами данными Вам необходимо дополнить модуль и реализовать соответствующие методы модуля io_base, позволяющий считывтаь данные из сответствующего типа файла (.txt, .png, .jpg). Для чтения текстовых данных необходимо реализовать классы-потомки базового класса IODataBase.

 Реализация методов read|save - техническая задача чтения/записи файлов, сами значения исходных файлов не изменяются в рамках выполнения этого задания.

Модуль checksum
 Для определения уникальности данных необходимо реализовать подсчет контрольной суммы.
 Контрольная сумма рассчитывается при помощи следующих алгоритмов:

 Алгоритм 1. 
 1. Данные изображения преобразуются в оттенок серого при помощи усреднения компонент RGB.
 2. Изображение разбивается на патчи в количестве 4х4 (размер каждого патча M/4, N/4).
 3. Для каждого патча:
    3.1. рассчитать среднее значение интенсивностей;
    3.2. Полученная характеристики бинаризуется: если среднее значение более 127 - 1, в ином случае - 0;
 4. Полученные 16 значений формируют бинарные разряды соответствующего бинарного числа (ключа). 

 Алгоритм 2.
 1. Рассчитывается среднее значение интенсивности пикселей данных.

Обе реализации - классы-наследники базового класса CMBase.

Модуль hash_data.
 - Формирование базы уникальных данных реализуется с помощью модуля hash_data.
 - Элемент данных (key, value), key - значение контрольной суммы, value - путь к данным (расположенным в папке dataset).
 - Необходимо определить магические операции добавления, удаления, получения элемента по ключу.
 - реализовать метод сохранения результата в файл ./output/hash_table.json
 - Метод __setitem__:  применять key_coder, переданного в __init__ при генерации ключа в том случае, если key = None или 0 при вызове __setitem__.

 Необходимо выполнить реализацию всех предложенных методов, возможно добавление вспомогательных методов и классов.
 Добавить базовую проверку исключений (некорректный тип файла, наличие/отсутствие файла, наличие/отсутствие элемента с соответствующим ключом, выход за границы набора данных).
